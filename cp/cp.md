这是一个java小游戏.
我想要实现的功能。
1.玩家可以选择多个人物。每个人物都有不同的技能。帮我构思一下。

人物与技能构思（可落地为数据或类）：
- 探险家：被动“发现遗迹”提升掉落概率；主动“勘探”显示隐藏路径/陷阱
- 猎人：被动“标记弱点”提升对精英伤害；主动“陷阱”减速/定身
- 祭司：被动“神谕”降低诅咒/负面效果；主动“净化”移除debuff
- 工匠：被动“拆解”回收材料更多；主动“机关破解”缩短开门/开箱时间
- 盗贼：被动“潜行”降低遇敌率；主动“速开”高几率不触发陷阱
- 战士：被动“硬化”减伤；主动“破墙”可开启隐藏区域

建议：每个角色包含基础属性（HP/ATK/DEF/速度/感知）、被动技能、主动技能（冷却/消耗）。

角色设计（已落地到 cp.json / ski.json）：

1) 探险家（探索/侦察）
   - 基础属性：HP120 / ATK22 / DEF14 / 速度18 / 感知28
   - 被动：发现遗迹（掉落+10%，隐藏路径触发+15%）
   - 主动：勘探（侦测下个节点风险，命中+10%持续2回合）

2) 猎人（爆发/控制）
   - 基础属性：HP110 / ATK26 / DEF12 / 速度24 / 感知20
   - 被动：标记弱点（对精英/首领伤害+10%）
   - 主动：束缚陷阱（80%ATK伤害，束缚1回合）

3) 祭司（治疗/净化）
   - 基础属性：HP130 / ATK18 / DEF16 / 速度16 / 感知22
   - 被动：神谕护佑（负面状态持续回合-1）
   - 主动：净化（移除1个debuff并回复15%最大HP）

4) 工匠（资源/解谜）
   - 基础属性：HP140 / ATK20 / DEF20 / 速度12 / 感知18
   - 被动：拆解回收（材料收益+20%）
   - 主动：机关破解（解除陷阱或获得额外材料）

5) 盗贼（机动/收益）
   - 基础属性：HP105 / ATK23 / DEF10 / 速度30 / 感知24
   - 被动：潜行（遇敌率-15%）
   - 主动：速开（开箱不触发陷阱概率+70%，成功额外金币）

6) 战士（坦度/稳定）
   - 基础属性：HP160 / ATK24 / DEF22 / 速度14 / 感知14
   - 被动：硬化（受到伤害-8%）
   - 主动：破墙（120%ATK伤害，破除护盾/障碍效果）
